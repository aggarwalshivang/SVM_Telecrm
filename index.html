Setup Instructions & Schema

1. Supabase Setup

Run this SQL in your Supabase project to create the single messages table and view:

CREATE TYPE message_direction AS ENUM ('inbound', 'outbound');
CREATE TYPE message_status AS ENUM ('sent', 'delivered', 'read');

CREATE TABLE messages (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  phone VARCHAR(20) NOT NULL,
  name VARCHAR(255),
  direction message_direction NOT NULL,
  body TEXT NOT NULL,
  media_url TEXT,
  timestamp TIMESTAMPTZ DEFAULT NOW()
);

CREATE OR REPLACE VIEW contacts_view AS
SELECT phone as id, phone, MAX(name) as name, MAX(timestamp) as updated_at, 0 as unread_count
FROM messages GROUP BY phone;

ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anon access" ON messages FOR ALL USING (true);

2. n8n Setup

Your n8n workflow must accept POST requests at the Webhook URL with a body containing:

{ "to": "919876543210", "name": "Alice", "message": "Text" }

It should return { "success": true } upon queuing/sending the message.
